<Update label="1.1.0" description="November 25, 2025">
RenderFlow 1.1 is here with an updated look, granular access control and software analytics.

**Updates:**
- New granular permission settings per user group
- Software analytics to review and compare apps and plugins across all computers
- User interface improvements, updated table, state colors, and quick menus
- New UI for dedicated render nodes
- Added option to disable automatic window close when the user is idle for 15 minutes
- Dark mode is now the default
- Added tiled rendering option to Scene Manager jobs
- Spawner mode can now be scheduled
- Added filter by tag and filter by pool

**Fixes:**
- [3ds Max] Fixed submit issue when using Corona multimap in the environment
- [3ds Max] Fixed Corona render crash during file load ("An exception occurred while rendering with 3ds Max") that caused tasks to be incorrectly marked as completed
- [3ds Max] Fixed issue with asset check falsely reporting missing textures
</Update>

<Update label="1.0.9" description="October 30, 2025">
This release introduces a new pool system, tiled rendering, task filtering, and numerous improvements and bug fixes.

**Updates:**
- New pool system
- Tiled rendering
- Updated benchmark database with the latest CPU/GPU values
- Added average frame time to the job details
- DR nodes are now released when there is no more work
- Changed "Elapsed Time" to "Total Render Time" for better clarity
- Node names are now sorted alphanumerically
- New default node assignment priority (node, workstation, server, and V-Ray CPU benchmark)
- Added access to render settings when 3ds Max file is browsed
- Added option to bulk tag computers
- Option to refresh the software list for multiple computers at once
- Option to save render elements to separate folders for 3ds Max
- Added support for Cinema 4D 2026
- Added support for Cinema 4D take system
- Added support for V-Ray for Blender
- Added task filtering and multi page selection
- Added an input to select a list or range of tasks
- Added option to open the completed frames
- Added support for Finish & Suspend through the API and CLI

**Fixes:**
- Corrected the missing Scene Manager setups dialog, "…you may need to open and save your scene again…"
- Users from the node are now removed, and the workstation username will always refresh based on the currently signed-in user
- During discovery the server alias will be shown if available
- If host version isn't specified, the latest available will be used instead of the oldest
- Fixed an issue where EXR and other format settings weren't applied during rendering
- Fixed an issue when Scene Manager settings are not available in the submitter
- Fixed issue with Cinema 4D and Blender submitter plugin not working in some cases
- Fixed issue with Blender output path not being read correctly
- Fixed issue with incorrect usernames on jobs
- Fixed issue where the scheduled date wasn't refreshing on job templates
- Fixed issue with 3ds Max/Corona saving black frames and completing the task
- Fixed duplicate job submissions from clicking the submit button multiple times
- Fixed issue where tasks in "Finishing" state couldn't be reactivated
- Fixed output save dialog always resetting to JPG
- Fixed issue with 3ds Max output format settings when browsing a file
- Fixed issues with scheduler time editing not working
- Fixed issue where some blacklist processes weren't considered in auto node mode
- Fixed automatic log service and job cleanup
- Fixed Fusion version detection, we now recognize major and patch numbers as well
- Fixed interruptible option not working in multi-selection
- Fixed issue where different versions could still connect during initial connection
- Fixed issue that prevented log collection when there's a space in the repository folder
</Update>

- Fixed issues with sorting in the nodes and jobs tables
- Fixed a problem where restore job wasn't moving job files to the active repository folder
- Archived jobs option is now always available, even when no jobs are submitted

- Archived job search now includes results based on username
- Added option to delete archived jobs
- Fixed issue where Estimated Finish Date and other properties were being truncated

- Fixed several issues with job start time and render time calculations
- Fixed issue with plugin discovery when 3ds Max had never been started

<Update label="1.0.1" description="June 2, 2025">
Hotfix 1 with improvements, bug fixes and a few new features.

**Updates:**
- New Cinema 4D job type
- New CPU Affinity slider to control thread count during rendering
- Added Finished and Suspend option to node quick menu (top right corner)
- Camera list is now sorted in alphabetical order
- [Blender] New "Submit to RenderFlow" toolbar button
- [3ds Max] Cameras and output format settings are now saved to metadata and will be available when browsing files in the submitter
- Added "Submit to RenderFlow" to Windows context menu for .max, .blend, .c4d, and .vrscene files
- New drag-and-drop feature: Drop files onto Jobs UI to open submitter with pre-filled properties
- New CLI tool (rfcli.exe) to submit jobs and retrieve node, job, and task-related data

**Fixes:**
- [3ds Max] Added support for plugin discovery in ADSK_3DSMAX_PLUGINS_ADDON_DIR environment variable
- [3ds Max] Fixed "Submit to RenderFlow" toolbar button in 3ds Max 2023 and earlier
- [3ds Max] Increased 3ds Max startup limit to 3 minutes and added optional environment variable (RF_JOB_PROCESS_TIMEOUT) to override it
- [Blender] Fixed job preparation and output issues
- Added VC_redist.x64.exe to the installer
- Fixed false positive Offline status caused by Clock Skew
- Fixed dark mode background for some components
</Update>
