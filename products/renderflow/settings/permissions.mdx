Use groups and policies to control what each user can do in RenderFlow.

## Groups

Groups let you organize users and assign permissions. Two groups exist by default:

- **Admin** — Full access to all features
- **User** — Standard access

<Frame caption="Groups settings page">
  <img src="https://placehold.co/900x300?text=Groups+Settings" alt="Groups settings" />
</Frame>

### Creating a group

1. Go to **Settings → Groups**
2. Click **Add Group**
3. Enter a name for the group

### Deleting a group

When you delete a group, all users in that group are moved to the default User group.

## Policies

Policies control what each group can do. This lets you tailor RenderFlow access for different team roles.

### Example team setup

<AccordionGroup>
  <Accordion title="3D Artists">

    - Can activate/suspend their own workstation
    - Cannot restart or shutdown other nodes
    - Cannot access system settings
  </Accordion>
  <Accordion title="Group Leaders">
    - Can manage jobs from their team (same group)
    - Can suspend/resume nodes in their group
    - Can view statistics and analytics
    - Cannot restart or shutdown nodes
    - Cannot change system-wide settings
  </Accordion>
  <Accordion title="Admins">
    - Full access to all jobs and nodes
    - Can restart and shutdown any node
    - Can enable maintenance mode
    - Can manage users, groups, and policies
    - Can configure system settings
  </Accordion>

### Job management permissions

| Permission | Description |
|------------|-------------|
| Create Job | Submit new jobs |
| Start Job | Start a suspended job |
| Stop Job | Stop a running job |
| Finish Job | Finish current task and stop |
| Reset Job | Reset job to initial state |
| Delete Job | Remove a job |
| Edit / Update Job | Change job settings and priority |
| Archive Job | Move job to archive |
| Restore Archived Job | Restore from archive |
| Delete Archived Job | Permanently delete archived job |

### Node management permissions

| Permission | Description |
|------------|-------------|
| Edit Node | Change node name, group, tags, or settings |
| Restart Node | Restart the computer |
| Shutdown Node | Power off the computer |
| Suspend Node | Prevent node from working on jobs |
| Reserve Node | Mark node as reserved |
| Resume Node | Allow node to work on jobs again |
| Wakeup Node | Power on using Wake-on-LAN |
| Delete Node | Remove node from the system |
| Finish Task & Suspend | Complete current task then suspend |
| Change Pool | Move node to a different pool |
| Change Mode | Switch between Node and Workstation |
| Start/Stop Spawner | Control the spawner process |
| CPU Affinity | Configure CPU core allocation |
| Remote Desktop Access | Connect via RDP |

### System permissions

| Permission | Description |
|------------|-------------|
| Software Analytics | View installed software across nodes |
| View Statistics | Access system-wide statistics |
| View/Edit Scheduler | Access and modify scheduled events |
| Edit Sanity Checks | Create and modify sanity checks |
| Error Reporter | Collect diagnostic data for support |
| System Idle Mode | Configure automatic idle detection |
| Maintenance Mode | Enable system-wide maintenance |
| System Settings | Access admin settings |
| Manage API Keys | Create and revoke API keys |

### Permission levels

Some permissions have scope levels:

| Level | Description |
|-------|-------------|
| **None** | No access |
| **Own** | Only items created by the user |
| **Group** | Items from users in the same group |
| **Everyone** | All items |

Other permissions are simply enabled or disabled.
