System settings control farm-wide behavior. Access these from **Settings → System**.

## Maintenance mode

When enabled, maintenance mode prevents new work from starting across the entire farm:

- Users cannot submit new jobs
- Active tasks complete normally
- Nodes won't start new tasks after finishing current work

Use this for system-wide updates, debugging, or scheduled maintenance windows.

## Automatic node mode

Configure when workstations should automatically start rendering. When a computer meets all the specified criteria, it becomes available for tasks.

| Setting | Description |
|---------|-------------|
| **Idle time** | Minutes of inactivity before rendering starts |
| **CPU usage** | Maximum average CPU usage (over 5 minutes) |
| **RAM** | Minimum available RAM in GB |
| **Blacklisted processes** | Applications that prevent rendering when running |

When you return to work, RenderFlow detects activity and gracefully finishes the current task before suspending.

### Example configuration

For artist workstations that should render during lunch breaks:

- **Idle time:** 10 minutes
- **CPU usage:** 10%
- **RAM:** 21 GB
- **Blacklisted processes:** `3dsmax.exe`

This ensures rendering only starts when the artist is away and their 3D application is closed.
