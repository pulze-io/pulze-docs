## Before you begin

Review the [System Requirements](/products/renderflow/getting-started/requirements) and ensure your firewall and network are properly configured before proceeding with installation.

<Info>
**For IT Administrators:** Looking for silent installation, Windows Service deployment, or automated rollout? See [Silent Installation](/products/renderflow/getting-started/silent-installation).
</Info>

## Choosing your server machine

Select a computer on your network to act as the RenderFlow server. For optimal performance with multiple render nodes and workstations, we recommend running the server on a dedicated machine that remains available 24/7.

For smaller studios or lighter workloads, your primary workstation can double as the server.

## Running the installer

Download the latest RenderFlow installer from [pulze.io](https://pulze.io) and run it on your server machine.

After installation, RenderFlow prompts you to select an operating mode. Choose **Server** to configure this machine as the central coordinator for your render farm.

### License configuration

Upon successful authentication, RenderFlow retrieves your available licenses. Select your preferred license management approach:

- **Automatic** — RenderFlow manages license allocation across your farm
- **Manual** — You control which licenses are assigned to specific nodes

### Repository configuration

Define a **shared network folder** (Repository) that all workstations and render nodes can access. RenderFlow uses this location to store:

- Submitted scene files
- Render logs
- Thumbnails and previews
- Job-related metadata

## Adding workstations and render nodes

With the server running, install RenderFlow on your remaining machines.

### Selecting the appropriate mode

During setup, choose the mode that matches the machine's primary purpose:

| Mode | Default Status | Recommended For |
|------|----------------|-----------------|
| **Node** | Idle | Dedicated render machines that should immediately accept jobs |
| **Workstation** | Suspended | Artist machines that render only when explicitly enabled |

You can adjust these settings at any time, but selecting the appropriate mode during installation helps organize your farm from the start.

<Frame caption="Server connection configuration for nodes">
  <img src="https://placehold.co/700x400?text=Node+Setup" alt="Node setup" />
</Frame>

### Connecting to the server

Enter the **IP address** or **hostname** of your RenderFlow server. Alternatively, click **Discover** to automatically locate the server on your network.

Once connected, the machine registers with the server and appears in your node list.

## Single machine setup

RenderFlow provides value even on a single workstation, functioning as a sophisticated batch rendering tool for managing multiple jobs locally.

To enable single-machine rendering:

1. Configure RenderFlow in **Server** mode
2. Enable **Node mode** in settings, or change your machine's status to **Idle**

This configuration allows your workstation to both submit and process render jobs.
