RenderFlow is an artist-friendly network rendering solution built for studios of all sizes. From small teams with a handful of machines to large production facilities with hundreds of render nodes, RenderFlow makes it easy to manage your render farm without the technical complexity.

## What is RenderFlow?

RenderFlow automates the entire network rendering workflow by automatically distributing render tasks across your available machines so artists can focus on creating instead of managing infrastructure.

## Architecture overview

RenderFlow uses a server-node architecture where one machine acts as the central **Server** that coordinates all rendering activity, while other machines connect as **Nodes** or **Workstations**.

<Frame caption="Architecture diagram showing server, nodes, and workstations">
  <img src="https://placehold.co/900x400?text=Architecture+Diagram" alt="RenderFlow architecture diagram" />
</Frame>

- **Server** — The central hub that manages the job queue, coordinates task distribution and stores the database. One server can manage hundreds of connected machines.
- **Node** — A dedicated render machine that connects to the server and processes assigned tasks. Nodes automatically report their available software, hardware capabilities, and current status.
- **Workstation** — An artist's machine that can submit jobs and optionally participate in rendering. Workstations start in a suspended state by default, so they only render when the artist chooses to contribute their machine.
