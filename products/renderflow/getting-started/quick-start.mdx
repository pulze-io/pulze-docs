This guide walks you through submitting your first render job. By the end, you'll understand the basic workflow that you'll use for all future jobs.

## Before you begin

Make sure you have:

- RenderFlow running with at least one node in **Idle** status
- A scene file ready to render (3ds Max, Blender, or Cinema 4D)
- The scene saved with all assets accessible via network paths

## Step 1: Start a new job

From the **Jobs** page, click the **Create** button in the top-right corner.

<Frame caption="Jobs page with Create button">
  <img src="https://placehold.co/900x400?text=Create+Button" alt="Jobs page with Create button" />
</Frame>

Alternatively, drag and drop your scene file directly onto the Jobs page — RenderFlow will detect the file type and start the submission process automatically.

## Step 2: Select a template

Choose a job template that matches your software and render engine. Templates come pre-configured with sensible defaults for common scenarios.

<Frame caption="Template selection screen">
  <img src="https://placehold.co/700x450?text=Template+Selection" alt="Template selection" />
</Frame>

For your first job, select the default template for your application (e.g., "3ds Max - V-Ray" or "Blender - Cycles").

## Step 3: Configure job settings

The submission wizard guides you through the essential settings:

<Frame caption="Job submission form">
  <img src="https://placehold.co/900x500?text=Job+Submission+Form" alt="Job submission form" />
</Frame>

### Basic settings

- **Job Name** — Give your job a descriptive name
- **Frame Range** — Specify which frames to render (e.g., `1-100` or `1,5,10-20`)
- **Output Path** — Where to save the rendered frames

### Output settings

- **Resolution** — Image dimensions (or use scene settings)
- **Output Format** — EXR, PNG, JPG, etc.

### Farm settings

- **Pool** — Select which group of nodes should process this job
- **Priority** — Set relative importance (Normal is fine for most jobs)

## Step 4: Review assets

The **Assets** tab shows all files your scene depends on — textures, references, caches, etc.

<Frame caption="Assets tab showing file dependencies">
  <img src="https://placehold.co/900x400?text=Assets+Tab" alt="Assets tab" />
</Frame>

RenderFlow checks that all assets are accessible from the render nodes. If any files show warnings, ensure they're saved to a network location all nodes can reach.

## Step 5: Run sanity checks

The **Sanity** tab validates your job before submission, checking for common issues:

- Missing assets or broken file paths
- Software version compatibility
- Output path accessibility

<Frame caption="Sanity check results">
  <img src="https://placehold.co/900x400?text=Sanity+Checks" alt="Sanity check results" />
</Frame>

Address any critical issues before proceeding. Warnings can often be ignored for test renders.

## Step 6: Submit

Click **Submit** to send your job to the farm. RenderFlow will:

1. Copy your scene to the repository (if configured)
2. Create tasks for each frame or frame range
3. Add the job to the queue

## Monitoring your job

After submission, your job appears in the Jobs list. Click on it to see detailed progress.

<Frame caption="Job in progress with tasks visible">
  <img src="https://placehold.co/900x500?text=Job+In+Progress" alt="Job in progress" />
</Frame>

### Understanding job status

| Status | Meaning |
|--------|---------|
| **Pending** | Waiting for available nodes |
| **Working** | Tasks are being processed |
| **Completing** | Final tasks finishing up |
| **Completed** | All tasks finished successfully |
| **Failed** | One or more tasks encountered errors |

### Task progress

Expand the job to see individual task progress:

- **Queued** — Assigned to a node, waiting to start
- **Working** — Currently rendering
- **Completed** — Finished successfully
- **Failed** — Encountered an error (click for details)

### Real-time updates

The progress bar updates in real-time as frames complete. You can also see:

- Which node is processing each task
- Estimated time remaining
- Render previews (when available)

## Troubleshooting first jobs

<AccordionGroup>
  <Accordion title="Job stuck in Pending">
    - Check the Nodes page — are any nodes in **Idle** status?
    - Verify nodes have the required software installed
    - Ensure the job's pool has nodes assigned to it
  </Accordion>
  <Accordion title="Tasks failing">
    - Click the failed task to view the error log
    - Common issues: missing assets, licensing errors, out of memory
    - Try resetting the failed task to retry
  </Accordion>
  <Accordion title="No output files">
    - Verify the output path is accessible from render nodes
    - Check that nodes have write permissions to the output folder
    - Review the task logs for write errors
  </Accordion>
</AccordionGroup>
