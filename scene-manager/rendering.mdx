The rendering modules bring together everything you've configured in your [setups](/scene-manager/setups) - [cameras](/scene-manager/camera-and-framing), [lighting](/scene-manager/lighting), [scene visibility](/scene-manager/scene-control) - and define how the final image is produced. These modules control where renders are saved, what quality settings to use, which render elements to include, and how post-production is applied.

## Render Output

You can choose between multiple output modules depending on your render engine. By default you will always have the Render Output - Common which is mapped to the Render Setting > Common Tab > Render Output. In case of V-Ray you are able to use Render Output - Channels (V-Ray Separate Render Channels) and the Render Output - Raw module (V-Ray Raw Output).

<Frame caption="Render Output module with project folder, path rules, and file extension settings">
  <img src="/images/scene-manager/sm_module_render_output.png" alt="Render Output module with project folder, path rules, and file extension settings" />
</Frame>

For every camera or setup you can create outputs with different filenames and extensions by defining a project folder and adding in rules. If the generated path does not exist, Scene Manager will create it at render time. It's also possible to change the global format settings for jpg, png, tif, tga and exr extensions.

The output paths you define here are what get used when you [Bake Setups](/scene-manager/bake-setups) or submit to [RenderFlow](/renderflow/job-types/scene-manager).

## Render Settings

Pick from an initial set of the most important render settings for V-Ray, Corona and F-Storm and modify them per setup.

<Frame caption="Render Settings module with per-setup V-Ray, Corona, and FStorm controls">
  <img src="/images/scene-manager/sm_module_render_settings_custom.png" alt="Render Settings module with per-setup V-Ray, Corona, and FStorm controls" />
</Frame>

The Render Settings - Override module can take control over your Render Settings and modify almost every property per setup. Once enabled you have to define a default render preset by pressing the Fetch button. After that you can start modifying your settings in the Render Setting Dialog and by pressing Fetch you can "lock" down these changes per setup. You can see how many and which properties are changed.

<Frame caption="Render Settings Override module showing fetched property changes per setup">
  <img src="/images/scene-manager/sm_module_render_settings_override.png" alt="Render Settings Override module showing fetched property changes per setup" />
</Frame>

As an example, in case of V-Ray this module can be used to fine tune all kinds of performance/quality settings per setup. In Corona you could control all the post processing and bloom and glare properties per camera, and these settings will be also shown in the Corona VFB.

You can load 3ds Max Render Preset files (`*.rps`) for each setup. When switching between the setups, Scene Manager will first apply the render settings that you loaded, and after that it will overwrite it with all the other active settings that you have in each module ([Resolution](/scene-manager/camera-and-framing#resolution), Render Output, [Environment](/scene-manager/lighting#environment), etc.).

<Frame caption="Render Settings Preset module loading .rps files per setup">
  <img src="/images/scene-manager/sm_module_render_settings_preset.png" alt="Render Settings Preset module loading .rps render preset files per setup" />
</Frame>

## Render Elements

The Render Elements module will let you enable/disable the required elements per setup. As an example you can set multiple ZDepth passes with different max values for each camera, or you can control which Light Select pass to include in your final render to keep your final exr, cxr or vrimg file smaller.

<Frame caption="Render Elements module toggling individual render passes per setup">
  <img src="/images/scene-manager/sm_module_render_elements.png" alt="Render Elements module toggling individual render passes per setup" />
</Frame>

## Post-Production

The Post-Production module will allow you to control your V-Ray VFB Layers or the new Tone Mapping settings for Corona 8. Change the Tone Mapping or VFB Layers settings as you wish then press the fetch button. This will save the current state and whenever you activate the state again, Scene Manager will apply the VFB settings.

By pressing clear, Scene Manager will forget the settings and it will no longer apply the settings.

<Frame caption="Post-Production module with VFB layers and tone mapping controls">
  <img src="/images/scene-manager/sm_module_post_production.png" alt="Post-Production module with VFB layers and tone mapping controls" />
</Frame>

<Warning>
If you are using V-Ray you must manually enable "Save VFB color correction to RGB channel".
</Warning>

## Next steps

Once your render settings are configured, you can extend your workflow with [Scripting](/scene-manager/scripting) for custom automation. When you're ready to render, press the **Batch Render** button in the top toolbar to render all your setups with one click. You can also [Bake Setups](/scene-manager/bake-setups) to native scene states, or submit all setups directly to [RenderFlow](/renderflow/job-types/scene-manager).
